# Architecture Decision Records

Architecture Decision Records (ADRs) capture significant technical decisions — their context, the
decision itself, and the consequences — so the reasoning behind the codebase stays discoverable.

New records are numbered sequentially (`NNNN-title.md`) following the format of the existing ones. A
decision that is later reversed is kept for history with its status updated (e.g. `Superseded by
ADR-000X`).
